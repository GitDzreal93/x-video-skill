# AI漫剧全流程提示词专家

专业的 AI 漫剧提示词生成 Skill，采用模块化工作流，帮助用户从故事构思到完成专业分镜提示词。

## 核心特性

- **模块化工作流**：6个阶段独立可用，可任意组合
- **全流程模式**：一键从故事到完整 Excel 分镜表
- **平台通用**：提示词适配即梦/可灵/LibLib/Vidu等所有平台
- **双输出格式**：Markdown 表格即时查看 + Excel 专业交付
- **知识库驱动**：内置5000+提示词词汇库和300+电影风格库

## 触发条件

当用户提出以下需求时，立即使用本 Skill：
- "帮我做一个AI漫剧/AI短剧/AI动画"
- "生成漫剧分镜提示词"
- "写一个AI视频的剧本"
- "帮我设计漫剧的美术风格"
- "生成文生图/图生视频提示词"
- "帮我配音/生成音效/合成视频/后期制作"
- "漫剧:剧本 / 漫剧:美术 / 漫剧:分镜 / 漫剧:文生图 / 漫剧:图生视频 / 漫剧:生图 / 漫剧:生视频 / 漫剧:后期 / 漫剧:全流程"

## 工作流程总览

```
Phase 1: 剧本创作  →  Phase 2: 美术风格  →  Phase 3: 分镜拆解
                                                    ↓
Phase 5: 图生视频  ←  Phase 4: 文生图提示词  ←─────┘
         ↓
Phase 6: AI生图/生视频（火山引擎即梦API）
         ↓
    Phase 7: 全流程输出（Excel）
         ↓
Phase 8: 后期制作（配音/音效/BGM/字幕/合成）
```

每个阶段可独立调用，也可通过 `漫剧:全流程` 串联执行。

### 工作区（Workspace）

每个漫剧项目使用独立的工作区目录，避免多个项目的输出混在一起。

```
项目工作区/
├── 分镜数据.json           # 分镜脚本数据
├── 分镜表.xlsx             # Excel 交付物
├── output_images/          # AI 生成的图片
├── output_videos/          # AI 生成的视频 + 最终成片
└── output_audio/
    ├── tts/                # TTS 台词配音
    ├── sfx/                # 音效
    ├── subtitle.srt        # 字幕
    └── bgm.mp3             # BGM
```

所有脚本支持 `--workspace` 参数指定工作区路径：

```bash
# 指定工作区（推荐）
python3 scripts/generate_tts.py --file 分镜数据.json --workspace /path/to/项目/

# 不指定时，使用当前目录作为工作区
python3 scripts/generate_tts.py --file 分镜数据.json
```

创建新项目时，先建立工作区目录：

```bash
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_images"
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_videos"
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_audio/tts"
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_audio/sfx"
```

## Phase 1: 剧本创作（漫剧:剧本）

### 触发条件
用户说"帮我写剧本"、"我有一个故事想法"、"漫剧:剧本"

### 步骤1：收集需求

必须询问的信息（缺什么问什么，已有信息不要重复问）：

```markdown
1. **故事主题**：你想讲什么故事？（一句话概括）
2. **风格类型**：古风/现代/科幻/武侠/奇幻/校园/悬疑？
3. **目标时长**：短视频（15-60秒）？短片（1-5分钟）？
4. **角色设定**：有几个主要角色？他们的基本特征？
5. **情绪基调**：热血/治愈/悲伤/悬疑/搞笑/史诗？
6. **目标平台**：竖屏(9:16)/横屏(16:9)/电影宽屏(2.35:1)？
```

### 步骤2：创建剧本

基于收集的信息，按照以下结构输出剧本：

```markdown
## 剧本：[标题]

**风格**：[风格类型]
**时长**：[预估时长]
**画幅**：[画面比例]

### 角色设定
- 角色A：[外观描述 + 性格标签 + 标志性特征]
- 角色B：[外观描述 + 性格标签 + 标志性特征]

### 场景列表
- 场景1：[场景描述 + 时间 + 天气/光线]

### 剧本正文

**场景1：[场景名]**（[时间]，[光线/天气]）

[动作描述] 角色A [具体行为]。
角色A（[情绪状态]）：[台词内容]
[动作描述] 角色B [具体行为]。
角色B（[情绪状态]）：[台词内容]

**场景2：[场景名]**...
```

### 剧本创作原则
1. **开场即钩子**：前3秒必须有冲突/悬念/视觉冲击
2. **三幕式结构**：建置→对抗→解决
3. **人物立体化**：差异化标签 + 动机驱动行为
4. **用动作代替台词**：能用表情/肢体表达的，减少台词
5. **控制信息密度**：避免单镜头塞满过多动作或台词

## Phase 2: 美术风格设计（漫剧:美术）

### 触发条件
用户说"帮我设计风格"、"确定美术方向"、"漫剧:美术"

### 步骤1：确认风格方向

如果没有在 Phase 1 中确定，询问：
1. **视觉风格偏好**：国漫3D / 水墨武侠 / 赛博朋克 / 日漫二次元 / 电影写实 / 其他
2. **色调偏好**：暖色 / 冷色 / 对比色 / 指定色彩
3. **参考作品**：有无喜欢的视觉参考？

### 步骤2：输出风格指南

```markdown
## 美术风格指南

### 风格定位
- **整体风格**：[如：国漫3D + 古风宫斗]
- **参考调性**：[如：张艺谋式色彩对比 + 韦斯安德森对称构图]

### 色彩方案
- **主色调**：[如：暗红+金黄]
- **辅色调**：[如：墨绿+深紫]
- **点缀色**：[如：金色+白色]
- **色调随剧情变化**：[如：开场暖金→高潮暗红→结尾清冷蓝]

### 光影方案
- **主光源**：[如：烛火暖光，淡黄色调]
- **光效特征**：[如：摇曳光影，人物面部半明半暗]
- **特殊光效**：[如：逆光剪影用于离别场景]

### 构图偏好
- **主要构图**：[如：对称构图强调宫廷秩序]
- **景别分配**：[如：对话多中景近景，情感爆发多特写]

### 角色色卡
- **角色A**：[标志色彩+服饰描述+配饰]
- **角色B**：[标志色彩+服饰描述+配饰]
```

> 参考知识库：`references/visual-style-guide.md` 和 `references/movie-styles.md`

### 步骤3：生成角色三视图提示词

**必须为每个有台词的主要角色生成三视图提示词。**

#### 三视图提示词公式

```
[风格] + [角色全身详细描述] + 角色三视图/多角度展示 + [白色纯净背景] + 正面+左侧面+背面
```

#### 输出格式

```markdown
### 角色三视图

| 角色名 | 三视图文生图提示词 | 标志性特征 | 标志色 |
|--------|-------------------|-----------|--------|
| 角色A | [完整三视图提示词] | [便于跨镜头识别的特征] | [主色调] |
| 角色B | [完整三视图提示词] | [便于跨镜头识别的特征] | [主色调] |
```

#### 三视图生成要点
- 角色描述必须包含：面部特征、发型、服装（从头到脚）、体型、配饰、标志性表情
- 背景必须纯色（白色），方便后续抠图
- 三视图生成后，后续所有文生图提示词应描述"参考@角色A三视图"
- 配角/群演不需要三视图

> 参考知识库：`references/prompt-formulas.md` 中的「角色三视图提示词」章节

## Phase 3: 分镜脚本拆解（漫剧:分镜）

### 触发条件
用户说"帮我拆分镜"、"做分镜表"、"漫剧:分镜"

### 输入要求
需要已有：剧本（Phase 1）或 用户直接提供的剧情描述

### 拆分原则

1. **镜头时长**：每个镜头 1-3 秒（AI视频最佳时长）
2. **连贯性**：相邻镜头保留共享元素（同一道具/光线/背景）
3. **情绪递进**：通过景别变化配合情绪起伏
4. **色调统一**：引用 Phase 2 的风格指南
5. **台词分配**：每句台词分配到对应镜头，标注说话者

### 输出格式

```markdown
## 分镜脚本：[标题]

**场景**：[统一场景描述（风格+场景+光影+色调）]

| 镜号 | 画面内容 | 景别 | 音乐/音效 | 台词 | 时长 |
|------|----------|------|-----------|------|------|
| 1 | [具体画面描述] | [景别] | [声音设计] | [台词或"-"] | [Xs] |
| 2 | ... | ... | ... | ... | ... |
```

> 参考知识库：`references/storyboard-templates.md`

## Phase 4: 文生图提示词生成（漫剧:文生图）

### 触发条件
用户说"生成文生图提示词"、"画面的提示词"、"漫剧:文生图"

### 提示词公式

```
[风格] + [场景/背景] + [主体描述+行为] + [环境细节] + [光影效果] + [构图方式] + [景别]
```

### 生成规则

1. **风格前缀统一**：所有镜头使用相同的风格开头（来自 Phase 2）
2. **场景描述一致**：同场景的镜头共用场景描述
3. **主体描述具体**：写清角色外观、动作、表情
4. **自然语言连贯**：用逗号分隔的连贯自然语言，不用符号标记
5. **参考词汇库**：从 `references/prompt-library.md` 选取精准词汇

### 首帧/尾帧提示词

为每个镜头额外生成首帧和尾帧文生图提示词：

- **首帧公式**：`[风格] + [场景] + [角色在此镜头的起始状态/姿势] + [光影] + [构图] + [景别]`
- **尾帧公式**：`[风格] + [场景] + [角色在此镜头的结束状态/姿势] + [光影] + [构图] + [景别]`

#### 首帧/尾帧设计原则

1. **动作衔接**：前一镜头尾帧的动作 = 后一镜头首帧的动作起点
2. **位置连贯**：角色在画面中的位置前后一致
3. **情绪递进**：首帧情绪 → 视频过渡 → 尾帧情绪有变化
4. **共享元素**：相邻首尾帧保留相同背景/光影/道具

### 输出格式

为分镜表中每个镜头生成三条提示词（首帧+尾帧+代表画面）：

```markdown
| 镜号 | 首帧文生图提示词 | 尾帧文生图提示词 | 文生图提示词（代表画面） |
|------|-----------------|-----------------|----------------------|
| 1 | [镜头起始状态] | [镜头结束状态] | [镜头代表画面] |
| 2 | ... | ... | ... |
```

> 参考知识库：`references/prompt-formulas.md` 和 `references/prompt-library.md`

## Phase 5: 图生视频提示词生成（漫剧:图生视频）

### 触发条件
用户说"生成视频提示词"、"图生视频"、"漫剧:图生视频"

### 提示词公式

```
[动作描述] + [镜头运动] + [情绪/照明]
```

### 生成规则

1. **动作描述**：1-2个核心动作，简洁明确
2. **镜头运动**：从词汇库选取精确的运镜描述
3. **情绪+照明**：标注情感氛围和光线特征
4. **跨镜头连贯**：动作在镜头间有接续关系

### 输出格式

```markdown
| 镜号 | 图生视频提示词 |
|------|---------------|
| 1 | 皇上猛拍桌案，镜头微颤，愤怒情绪，淡黄光影聚焦手部 |
| 2 | ... |
```

> 参考知识库：`references/prompt-formulas.md` 和 `references/prompt-library.md`

## Phase 6: AI 生图与生视频（漫剧:生图 / 漫剧:生视频）

### 触发条件
用户说"生成图片"、"帮我生图"、"漫剧:生图"、"生成视频"、"帮我生视频"、"漫剧:生视频"

### 前置条件
- 已完成 Phase 1-5（至少有分镜JSON数据，包含文生图/图生视频提示词）
- 已配置火山引擎 AK/SK

### 步骤1：配置 API 凭证

首次使用需要配置火山引擎 AK/SK。

**检查凭证是否已配置：**
```bash
cat ~/.ai-comic-drama/config.json 2>/dev/null || echo "未配置"
```

**如果未配置，必须先询问用户提供 AK/SK：**
> 使用 AI 生图/生视频功能需要火山引擎的 Access Key 和 Secret Key。
> 请提供你的 AK 和 SK。
> 获取方式：https://console.volcengine.com/iam/keymanage

**用户提供后，保存凭证：**
```bash
mkdir -p ~/.ai-comic-drama
python3 -c "
import json
config = {'ak': '用户提供的AK', 'sk': '用户提供的SK'}
with open('$HOME/.ai-comic-drama/config.json', 'w') as f:
    json.dump(config, f, indent=2)
print('凭证已保存')
"
```

也可以通过环境变量或命令行参数提供：
```bash
export VOLCENGINE_AK="your_access_key"
export VOLCENGINE_SK="your_secret_key"
```

### 步骤2：文生图（角色三视图 + 分镜画面）

#### 单张生成

```bash
# 生成单张图片
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_image.py \
  --prompt "国漫3d风格，角色三视图..." \
  --output 三视图_角色A.png \
  --aspect 1:1
```

#### 批量生成（从分镜JSON）

```bash
# 批量生成所有图片（三视图 + 首帧 + 尾帧 + 代表画面）
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_image.py \
  --file 分镜数据.json \
  --output ./output_images \
  --aspect 16:9
```

批量模式会自动生成：
1. 每个角色的三视图（1:1 正方形）
2. 每个镜头的首帧、尾帧、代表画面（按指定画幅）

#### 画幅预设

| 画幅 | 尺寸 | 适用场景 |
|------|------|----------|
| 9:16 | 1080×1920 | 抖音/快手竖屏 |
| 16:9 | 1920×1080 | B站/YouTube横屏 |
| 1:1 | 1024×1024 | 正方形（三视图） |
| 2.35:1 | 2322×988 | 电影宽屏 |

### 步骤3：图生视频（图片 → 动态视频）

#### 单个生成

```bash

python3 ~/.claude/skills/ai-comic-drama/scripts/generate_video.py \
  --image 镜头001_首帧.png \
  --prompt "角色转身，镜头缓慢拉远，紧张氛围" \
  --output 镜头001.mp4 \
  --duration 3
```

#### 首帧+尾帧模式

```bash
# 指定首帧和尾帧，AI自动生成过渡动画
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_video.py \
  --image 镜头001_首帧.png \
  --last-frame 镜头001_尾帧.png \
  --prompt "角色从站立到坐下" \
  --output 镜头001.mp4
```

#### 批量生成

```bash
# 从分镜JSON + 图片目录批量生成视频
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_video.py \
  --file 分镜数据.json \
  --image-dir ./output_images \
  --output ./output_videos
```

## Phase 7: 全流程（漫剧:全流程）

### 触发条件
用户说"全流程制作"、"从头到尾"、"漫剧:全流程"

### 执行流程

1. **收集需求**（Phase 1 的步骤1）
2. **创作剧本**（Phase 1 的步骤2）→ 请用户确认
3. **设计风格 + 角色三视图**（Phase 2，含步骤3角色三视图）→ 请用户确认
4. **拆分分镜**（Phase 3）
5. **生成首帧/尾帧/文生图提示词**（Phase 4）
6. **生成图生视频提示词**（Phase 5）
7. **输出完整分镜表**（Markdown + Excel）
8. **AI生图/生视频**（Phase 6，用户选择是否执行）

### 最终输出：完整分镜表

#### 第一部分：角色三视图

```markdown
### 角色三视图

| 角色名 | 三视图文生图提示词 | 标志性特征 | 标志色 |
|--------|-------------------|-----------|--------|
| 角色A | [完整三视图提示词] | [特征] | [主色] |
| 角色B | [完整三视图提示词] | [特征] | [主色] |
```

#### 第二部分：完整分镜表

```markdown
## 完整分镜表：[标题]

**风格**：[风格]
**时长**：[总时长]
**画幅**：[画面比例]

| 拍摄场景 | 镜号 | 画面内容 | 景别 | 音乐/音效 | 台词 | 时长 | 首帧文生图提示词 | 尾帧文生图提示词 | 文生图提示词 | 图生视频提示词 |
|----------|------|----------|------|-----------|------|------|-----------------|-----------------|-------------|---------------|
| [场景] | 1 | ... | ... | ... | ... | ... | [起始状态] | [结束状态] | [代表画面] | [视频动作] |
```

## 交互规范

### 模式识别
- 用户只描述故事 → 启动 Phase 1（剧本），完成后询问是否继续
- 用户已有剧本 → 从 Phase 2 开始（美术风格）
- 用户只想要提示词 → 直接 Phase 4 或 Phase 5
- 用户想生成图片/视频 → Phase 6（AI生图/生视频）
- 用户说"全流程" → Phase 7 模式，串联所有阶段

### 确认节点
在以下节点必须等待用户确认后再继续：
1. 剧本完成后 → "剧本已生成，请确认是否满意？确认后继续设计美术风格。"
2. 美术风格完成后 → "风格方案已确定，请确认后开始分镜拆解。"
3. 分镜完成后 → "分镜表已生成，请确认后生成提示词。"

### 输出规范
- **Markdown 表格**：默认输出格式，直接在对话中展示
- **Excel 文件**：用户要求时，调用导出脚本生成
- **纯文本提示词**：可单独输出每条提示词，方便复制

### 质量检查
每个阶段输出前自查：
- [ ] 风格前缀是否一致
- [ ] 色调是否统一
- [ ] 角色描述是否连贯
- [ ] 相邻镜头是否有共享元素
- [ ] 每个镜头时长是否 ≤3s
- [ ] 台词是否分配到正确镜头
- [ ] 每个有台词的角色是否都有三视图提示词
- [ ] 三视图提示词是否包含完整的外观细节（面部/发型/服装/配饰）
- [ ] 每个镜头是否都有首帧和尾帧提示词
- [ ] 前一镜头尾帧的动作是否与后一镜头首帧自然衔接

## Phase 8: 后期制作（漫剧:后期）

### 触发条件
用户说"帮我配音"、"生成音效"、"合成视频"、"后期制作"、"漫剧:后期"

### 前置条件
- Phase 6 完成（视频文件已生成）
- 分镜 JSON 中包含 `dialogue` 和 `sound` 字段

### 步骤1：TTS 台词配音

使用 edge-tts（微软神经网络语音，免费）或豆包 TTS（音质更好）为台词生成配音。

```bash
# 免费模式（edge-tts）
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_tts.py \
  --file 分镜数据.json --workspace /path/to/项目/ --engine free

# 豆包模式（音质更好，需配置 API Key）
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_tts.py \
  --file 分镜数据.json --workspace /path/to/项目/ --engine doubao

# 强制重新生成
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_tts.py \
  --file 分镜数据.json --workspace /path/to/项目/ --engine doubao --force
```

角色语音配置：
- **AI执政官**：高冷沉稳 2.0（豆包）/ `zh-CN-XiaoxiaoNeural`（免费）
- **臻叔**：云舟 2.0 讲故事风格（豆包）/ `zh-CN-YunjianNeural`（免费）

豆包 TTS 配置：在 `~/.ai-comic-drama/config.json` 中设置 `doubao_tts_api_key`

输出：`output_audio/tts/镜头XXX_台词.mp3` + `生成报告.json`

### 步骤2：音效生成

基于本地音效库，将中文音效描述自动匹配并混音。

```bash
# 初始化音效库目录（首次使用）
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_sfx.py --init-library

# 批量生成音效
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_sfx.py \
  --file 分镜数据.json --workspace /path/to/项目/
```

音效库路径：`~/.ai-comic-drama/sfx_library/`，按 5 类组织：
- `ambient/` — 环境音（雨声、风声、脚步声）
- `mechanical/` — 机械音（金属撞击、摩擦、吱呀声）
- `electronic/` — 科技音（电子音、蜂鸣、启动声）
- `action/` — 动作音（蒸汽、心跳、键盘、电源键）
- `climax/` — 高潮/结局音（爆炸、碎裂、钢琴、鸟鸣）

用户需自行将 WAV/MP3 音效文件放入对应目录。

### 步骤3：字幕生成

从分镜 JSON 的台词和时间数据自动生成 SRT 字幕。

```bash
# 用 TTS 音频实际时长精确计时
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_subtitle.py \
  --file 分镜数据.json --workspace /path/to/项目/
```

### 步骤4：BGM 选择

用户自行提供 BGM 音频文件（如从 Pixabay/Mixkit 获取免费音乐），放入：
- `output_audio/bgm.mp3`

### 步骤5：视频合成

将视频片段 + TTS + 音效 + BGM + 字幕合成为最终成片。

```bash
# 使用工作区（自动定位所有素材）
python3 ~/.claude/skills/ai-comic-drama/scripts/assemble_video.py \
  --file 分镜数据.json --workspace /path/to/项目/

# 手动指定各路径
python3 ~/.claude/skills/ai-comic-drama/scripts/assemble_video.py \
  --file 分镜数据.json --workspace /path/to/项目/ \
  --video-dir ./output_videos/ \
  --tts-dir ./output_audio/tts/ \
  --sfx-dir ./output_audio/sfx/ \
  --bgm ./output_audio/bgm.mp3 \
  --subtitle ./output_audio/subtitle.srt \
  --output ./output_videos/最终成片.mp4

# 跳过 BGM 和字幕
python3 ~/.claude/skills/ai-comic-drama/scripts/assemble_video.py \
  --file 分镜数据.json --workspace /path/to/项目/ \
  --no-bgm --no-subtitle
```

合成流程包含 Doctor 检查：自动验证素材完整性、时长匹配，音频超长时自动变速不截断。
