# AI漫剧全流程提示词专家

专业的 AI 漫剧提示词生成 Skill，采用模块化工作流，帮助用户从故事构思到完成专业分镜提示词。

## 核心特性

- **模块化工作流**：6个阶段独立可用，可任意组合
- **全流程模式**：一键从故事到完整 Excel 分镜表
- **平台通用**：提示词适配即梦/可灵 3.0/LibLib/Vidu/海螺AI等所有平台
- **可灵 3.0 深度整合**：智能分镜、主体参考、音画同步、多语种方言、15s长镜头
- **双输出格式**：Markdown 表格即时查看 + Excel 专业交付
- **知识库驱动**：内置5000+提示词词汇库和300+电影风格库

## 工作流程总览

```
Phase 1: 剧本创作  →  Phase 2: 美术风格  →  Phase 3: 分镜拆解
                                                    ↓
Phase 5: 图生视频  ←  Phase 4: 文生图提示词  ←─────┘
         ↓
Phase 6: AI生图/生视频（即梦API / 可灵3.0 / 海螺AI）
         ↓
    Phase 7: 全流程输出（Excel）
         ↓
Phase 8: 后期制作（配音/音效/BGM/字幕/合成）
```

每个阶段可独立调用，也可通过 `漫剧:全流程` 串联执行。

> **可灵 3.0 深度整合**：本 Skill 内置可灵视频 3.0 模型专属提示词体系，涵盖智能分镜/自定义分镜、主体参考、音画同步、多语种方言、原生文字、15s长镜头等能力，详见 Phase 4-6 对应章节。

### 工作区（Workspace）

每个漫剧项目使用独立的工作区目录，避免多个项目的输出混在一起。

```
项目工作区/
├── 分镜数据.json           # 分镜脚本数据
├── 分镜表.xlsx             # Excel 交付物
├── output_images/          # AI 生成的图片
├── output_videos/          # AI 生成的视频 + 最终成片
└── output_audio/
    ├── tts/                # TTS 台词配音
    ├── sfx/                # 音效
    ├── subtitle.srt        # 字幕
    └── bgm.mp3             # BGM
```

所有脚本支持 `--workspace` 参数指定工作区路径：

```bash
# 指定工作区（推荐）
python3 scripts/generate_tts.py --file 分镜数据.json --workspace /path/to/项目/

# 不指定时，使用当前目录作为工作区
python3 scripts/generate_tts.py --file 分镜数据.json
```

创建新项目时，先建立工作区目录：

```bash
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_images"
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_videos"
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_audio/tts"
mkdir -p "/Volumes/dz/ai_video/我的新项目/output_audio/sfx"
```

## Phase 1: 剧本创作（漫剧:剧本）

### 触发条件
用户说"帮我写剧本"、"我有一个故事想法"、"漫剧:剧本"

### 步骤1：收集需求

必须询问的信息（缺什么问什么，已有信息不要重复问）：

```markdown
1. **故事主题**：你想讲什么故事？（一句话概括）
2. **风格类型**：古风/现代/科幻/武侠/奇幻/校园/悬疑？
3. **目标时长**：短视频（15-60秒）？短片（1-5分钟）？
4. **角色设定**：有几个主要角色？他们的基本特征？
5. **情绪基调**：热血/治愈/悲伤/悬疑/搞笑/史诗？
6. **目标平台**：竖屏(9:16)/横屏(16:9)/电影宽屏(2.35:1)？
```

### 步骤2：创建剧本

基于收集的信息，按照以下结构输出剧本：

```markdown
## 剧本：[标题]

**风格**：[风格类型]
**时长**：[预估时长]
**画幅**：[画面比例]

### 角色设定
- 角色A：[外观描述 + 性格标签 + 标志性特征]
- 角色B：[外观描述 + 性格标签 + 标志性特征]

### 场景列表
- 场景1：[场景描述 + 时间 + 天气/光线]

### 剧本正文

**场景1：[场景名]**（[时间]，[光线/天气]）

[动作描述] 角色A [具体行为]。
角色A（[情绪状态]）：[台词内容]
[动作描述] 角色B [具体行为]。
角色B（[情绪状态]）：[台词内容]

**场景2：[场景名]**...
```

### 剧本创作原则
1. **开场即钩子**：前3秒必须有冲突/悬念/视觉冲击
2. **三幕式结构**：建置→对抗→解决
3. **人物立体化**：差异化标签 + 动机驱动行为
4. **用动作代替台词**：能用表情/肢体表达的，减少台词
5. **控制信息密度**：避免单镜头塞满过多动作或台词

## Phase 2: 美术风格设计（漫剧:美术）

### 触发条件
用户说"帮我设计风格"、"确定美术方向"、"漫剧:美术"

### 步骤1：确认风格方向

如果没有在 Phase 1 中确定，询问：
1. **视觉风格偏好**：国漫3D / 水墨武侠 / 赛博朋克 / 日漫二次元 / 电影写实 / 其他
2. **色调偏好**：暖色 / 冷色 / 对比色 / 指定色彩
3. **参考作品**：有无喜欢的视觉参考？

### 步骤2：输出风格指南

```markdown
## 美术风格指南

### 风格定位
- **整体风格**：[如：国漫3D + 古风宫斗]
- **参考调性**：[如：张艺谋式色彩对比 + 韦斯安德森对称构图]

### 色彩方案
- **主色调**：[如：暗红+金黄]
- **辅色调**：[如：墨绿+深紫]
- **点缀色**：[如：金色+白色]
- **色调随剧情变化**：[如：开场暖金→高潮暗红→结尾清冷蓝]

### 光影方案
- **主光源**：[如：烛火暖光，淡黄色调]
- **光效特征**：[如：摇曳光影，人物面部半明半暗]
- **特殊光效**：[如：逆光剪影用于离别场景]

### 构图偏好
- **主要构图**：[如：对称构图强调宫廷秩序]
- **景别分配**：[如：对话多中景近景，情感爆发多特写]

### 角色色卡
- **角色A**：[标志色彩+服饰描述+配饰]
- **角色B**：[标志色彩+服饰描述+配饰]
```

> 参考知识库：`references/visual-style-guide.md` 和 `references/movie-styles.md`

### 步骤3：生成角色三视图提示词

**必须为每个有台词的主要角色生成三视图提示词。**

#### 三视图提示词公式

```
[风格] + [角色全身详细描述] + 角色三视图/多角度展示 + [白色纯净背景] + 正面+左侧面+背面
```

#### 输出格式

```markdown
### 角色三视图

| 角色名 | 三视图文生图提示词 | 标志性特征 | 标志色 |
|--------|-------------------|-----------|--------|
| 角色A | [完整三视图提示词] | [便于跨镜头识别的特征] | [主色调] |
| 角色B | [完整三视图提示词] | [便于跨镜头识别的特征] | [主色调] |
```

#### 三视图生成要点
- 角色描述必须包含：面部特征、发型、服装（从头到脚）、体型、配饰、标志性表情
- 背景必须纯色（白色），方便后续抠图
- 三视图生成后，后续所有文生图提示词应描述"参考@角色A三视图"
- 配角/群演不需要三视图

> 参考知识库：`references/prompt-formulas.md` 中的「角色三视图提示词」章节

## Phase 3: 分镜脚本拆解（漫剧:分镜）

### 触发条件
用户说"帮我拆分镜"、"做分镜表"、"漫剧:分镜"

### 输入要求
需要已有：剧本（Phase 1）或 用户直接提供的剧情描述

### 拆分原则

1. **镜头时长**：每个镜头 1-3 秒（AI视频最佳时长）
2. **连贯性**：相邻镜头保留共享元素（同一道具/光线/背景）
3. **情绪递进**：通过景别变化配合情绪起伏
4. **色调统一**：引用 Phase 2 的风格指南
5. **台词分配**：每句台词分配到对应镜头，标注说话者

### 输出格式

```markdown
## 分镜脚本：[标题]

**场景**：[统一场景描述（风格+场景+光影+色调）]

| 镜号 | 画面内容 | 景别 | 音乐/音效 | 台词 | 时长 |
|------|----------|------|-----------|------|------|
| 1 | [具体画面描述] | [景别] | [声音设计] | [台词或"-"] | [Xs] |
| 2 | ... | ... | ... | ... | ... |
```

### 可灵 3.0 多镜头叙事

可灵 3.0 模型支持两种多镜头视频生成方式（需在可灵平台开启「智能分镜」开关）：

#### 方式一：智能分镜（AI 自动规划）

只需在 Prompt 中完整描述多场景、多角色的完整剧情，可灵 3.0 会自动解析场景流转、调度景别与机位。支持对话正反打、台词跨分镜、画外音等高级视听语言。

**智能分镜 Prompt 公式：**
```
[场景设定]，[角色A]做某事，说"台词"，镜头切换/景别变化，[角色B]做某事，说"台词"，镜头运动/场景变化...
```

**示例：**
> 欧洲别墅户外露台场景，铺着蓝白格纹桌布的餐桌旁，年轻白人女性穿蓝白条纹短袖衬衫、卡其色短裤，赤脚坐着，对面是穿白色 T 恤的年轻白人男性，镜头推进，女性晃着玻璃杯里的果汁，目光望向远处的树林，说"These trees will turn yellow in a month, won't they?"，镜头特写男性低着头说，"but they'll be green again next summer."，然后女性转头，笑着看向对面的男性，说，"Are you always this optimistic? Or just about summer?"，然后男性抬起头，看着女生说，"Only about summers with you."

#### 方式二：自定义分镜（精准控制每个镜头）

在开启「智能分镜」的前提下，通过 `shot N, [景别/角度] of [主体] + [动作/场景], [运镜], [风格描述]` 格式逐个定义每个镜头。

**自定义分镜 Prompt 公式：**
```
shot 1, [景别+角度] of [主体/场景], [镜头运动], [风格/氛围]
shot 2, [景别+角度] of [主体/场景], [镜头运动], [风格/氛围]
...
```

**示例：**
> shot 1, profile shot of black man driving a truck, cinematic handheld
> shot 2, frontal macro shot of black man driving a truck, cinematic handheld
> shot 3, macro shot of hands on the steering wheel, cinematic handheld
> shot 4, macro shot of a weathered picture of a young black child laying on the passenger side seat, cinematic handheld

**中文自定义分镜示例：**
> 镜头1-后远景低角度跟拍，骑手向前驶去
> 镜头2-侧方低角度近景，特写摩托车车轮
> 镜头3-骑手第一人称主观视角，前方是摩托车车把和表盘
> 镜头4-正面中景迎向摩托跟拍，骑手头盔正对镜头

> 参考知识库：`references/storyboard-templates.md`

## Phase 4: 文生图提示词生成（漫剧:文生图）

### 触发条件
用户说"生成文生图提示词"、"画面的提示词"、"漫剧:文生图"

### 提示词公式

```
[风格] + [场景/背景] + [主体描述+行为] + [环境细节] + [光影效果] + [构图方式] + [景别]
```

### 生成规则

1. **风格前缀统一**：所有镜头使用相同的风格开头（来自 Phase 2）
2. **场景描述一致**：同场景的镜头共用场景描述
3. **主体描述具体**：写清角色外观、动作、表情
4. **自然语言连贯**：用逗号分隔的连贯自然语言，不用符号标记
5. **参考词汇库**：从 `references/prompt-library.md` 选取精准词汇

### 首帧/尾帧提示词

为每个镜头额外生成首帧和尾帧文生图提示词：

- **首帧公式**：`[风格] + [场景] + [角色在此镜头的起始状态/姿势] + [光影] + [构图] + [景别]`
- **尾帧公式**：`[风格] + [场景] + [角色在此镜头的结束状态/姿势] + [光影] + [构图] + [景别]`

#### 首帧/尾帧设计原则

1. **动作衔接**：前一镜头尾帧的动作 = 后一镜头首帧的动作起点
2. **位置连贯**：角色在画面中的位置前后一致
3. **情绪递进**：首帧情绪 → 视频过渡 → 尾帧情绪有变化
4. **共享元素**：相邻首尾帧保留相同背景/光影/道具

### 输出格式

为分镜表中每个镜头生成三条提示词（首帧+尾帧+代表画面）：

```markdown
| 镜号 | 首帧文生图提示词 | 尾帧文生图提示词 | 文生图提示词（代表画面） |
|------|-----------------|-----------------|----------------------|
| 1 | [镜头起始状态] | [镜头结束状态] | [镜头代表画面] |
| 2 | ... | ... | ... |
```

> 参考知识库：`references/prompt-formulas.md` 和 `references/prompt-library.md`

## Phase 5: 图生视频提示词生成（漫剧:图生视频）

### 触发条件
用户说"生成视频提示词"、"图生视频"、"漫剧:图生视频"

### 提示词公式

```
[动作描述] + [镜头运动] + [情绪/照明]
```

### 生成规则

1. **动作描述**：1-2个核心动作，简洁明确
2. **镜头运动**：从词汇库选取精确的运镜描述
3. **情绪+照明**：标注情感氛围和光线特征
4. **跨镜头连贯**：动作在镜头间有接续关系

### 输出格式

```markdown
| 镜号 | 图生视频提示词 |
|------|---------------|
| 1 | 皇上猛拍桌案，镜头微颤，愤怒情绪，淡黄光影聚焦手部 |
| 2 | ... |
```

## 可灵 3.0 专属提示词体系

可灵视频 3.0 模型新增多项 AI 视频生成能力，需在 Prompt 中使用特定格式触发。本体系与通用提示词公式配合使用。

### 1. 音画同步与多角色指代

可灵 3.0 支持原生音画同步，在 Prompt 中标注角色台词时，模型自动解析角色与台词的对应关系，支持三人及以上的多角色指代。

**Prompt 公式：**
```
[场景设定]，[角色A描述]（[语气/情绪]，[语种/口音]）："[台词内容]"，[动作描述]。[角色B描述]（[语气/情绪]）："[台词内容]"
```

**示例（多角色对话）：**
> 居家环境，背景有轻微的客厅空调出风音，贴合写实日常。妈妈（轻声感慨，语气惊讶）：Wow, I didn't expect this plot at all. 爸爸（低嗓附和，语气平淡）：Yeah, it's totally unexpected, never thought that would happen. 男孩（语气雀跃）：It's the best twist ever! 女孩（跟着点头，语气激动）：I can't believe they did that!

**示例（三人及以上指代）：**
> 一位女老师和三个学生站在教室里，镜头固定，女老师说："我们不说，包的，但可以说"，这时说话人切换为穿着校服的女学生，女学生举起单手握拳说："志在必得"，这时说话人切换为穿着从第一位男学生，男学生微笑着说："万无一失"，这时说话人切换为穿着从站在最右边的男学生，男学生举起双手说："稳操胜券"

### 2. 多语种与方言/口音

可灵 3.0 支持中、英、日、韩、西五种语种，及方言口音。在角色台词后标注 `（语种/口音）` 即可触发。

**支持的语种：** 中文、英文、日文、韩文、西班牙文
**支持的中文口音：** 东北口音、北京口音、台湾地区口音、粤语、四川话等
**支持的英文口音：** 美式口音、英式口音、印度口音等

**Prompt 公式：**
```
[角色]（[语种/口音]，[语气]）："[台词]"
```

**示例（韩语）：**
> 韩国高中屋顶场景下，背景有远处的城市灯光与轻微的风声，星星在夜空闪烁。女主靠着栏杆发呆，男主拿着两罐可乐走来，递给女生，女主接过可乐打开。男主（语气轻松，韩语）："숙제 다 했어? 왜 여기 있어?" 女主（叹气，韩语）："시험이 너무 무서워." 男主（温柔，韩语）："걱정 마, 넌 잘할 거야."

**示例（西班牙语+中文游客）：**

### 3. 图生视频 + 主体参考

可灵 3.0 在图生视频基础上新增主体绑定能力，通过上传参考图/视频绑定主体，确保跨镜头主角始终如一。视觉主体 + 音色可同步绑定。

**主体参考 Prompt 公式（图生视频时附带主体参考图）：**
> 上传首帧图 + 上传主体参考图（绑定已创建的主体）
> Prompt: [场景延续] + [动作描述] + [镜头运动] + [情绪氛围]

**示例：**
> 真实职场质感，长镜头一镜到底无切换，全程中景平稳跟拍职场女性，镜头与人物同步移动：人物行走时镜头同步跟随，人物停顿时镜头即刻静止，动作自然连贯、镜头运镜流畅。女性向前走出电梯，身后电梯门缓慢自然闭合；她步入办公区，用手摘下墨镜，随手放进随身通勤包，向沿途擦肩而过的同事自然点头问好...

**主体一致性核心原则：**
- 角色类主体：上传 2-4 张参考图，或上传/录制人物视频
- 主体参考可与音色绑定：创建主体时上传音频或指定音色
- 已绑定音色的主体，不建议在 Prompt 中额外设置音色
- 主体参考 + 首帧图双重绑定效果最佳

### 4. 原生文字能力

**触发方式：** 在 Prompt 中描述文字内容，或在参考图中保留文字，模型自动识别并保持。

**示例（广告场景）：**

### 5. 15s 长镜头生成

可灵 3.0 支持 3-15 秒灵活时长生成，可承载更复杂的动作逻辑与情节变化。

**长镜头 Prompt 要点：**
- 按时间线组织：`第 X 秒，[画面内容]，[镜头运动]，[情绪氛围]`
- 明确标注总时长和镜头类型
- 时间分段描述动作变化

**示例：**

>
> 这是一个 15 秒的电影级长镜头。一镜到底，无剪辑转场。

### 视频生成引擎

支持多种图生视频引擎，通过 `--engine` 参数切换：

| 引擎 | 参数 | 原理 | 前提条件 | 核心能力 |
|------|------|------|----------|----------|
| 即梦AI | `--engine jimeng`（默认） | 火山引擎 API 调用 | AK/SK | 文生图/图生视频/首尾帧 |
| 可灵 3.0 | `--engine kling` | 可灵 API / 平台操作 | 可灵平台账号 | 智能分镜/主体参考/音画同步/多语种方言/15s长镜头/原生文字 |
| 海螺AI | `--engine hailuo` | CDP浏览器自动化 | Chrome CDP + 登录态 | 图生视频/首尾帧 |

**可灵 3.0 使用注意事项：**
- 可灵 3.0 支持「音画同步」和「无音画同步」两种模式
- 智能分镜和自定义分镜需在可灵平台打开对应开关

## 交互规范

### 模式识别
- 用户只描述故事 → 启动 Phase 1（剧本），完成后询问是否继续
- 用户已有剧本 → 从 Phase 2 开始（美术风格）
- 用户只想要提示词 → 直接 Phase 4 或 Phase 5
- 用户想生成图片/视频 → Phase 6（AI生图/生视频）
- 用户说"全流程" → Phase 7 模式，串联所有阶段

### 确认节点
在以下节点必须等待用户确认后再继续：
1. 剧本完成后 → "剧本已生成，请确认是否满意？确认后继续设计美术风格。"
2. 美术风格完成后 → "风格方案已确定，请确认后开始分镜拆解。"
3. 分镜完成后 → "分镜表已生成，请确认后生成提示词。"

### 输出规范
- **Markdown 表格**：默认输出格式，直接在对话中展示
- **Excel 文件**：用户要求时，调用导出脚本生成
- **纯文本提示词**：可单独输出每条提示词，方便复制

### 质量检查
每个阶段输出前自查：
- [ ] 风格前缀是否一致
- [ ] 色调是否统一
- [ ] 角色描述是否连贯
- [ ] 相邻镜头是否有共享元素
- [ ] 每个镜头时长是否 ≤3s
- [ ] 台词是否分配到正确镜头
- [ ] 每个有台词的角色是否都有三视图提示词
- [ ] 三视图提示词是否包含完整的外观细节（面部/发型/服装/配饰）
- [ ] 每个镜头是否都有首帧和尾帧提示词
- [ ] 前一镜头尾帧的动作是否与后一镜头首帧自然衔接
- [ ] **（可灵 3.0 智能分镜）** 多镜头叙事是否包含清晰的场景流转和景别变化
- [ ] **（可灵 3.0 自定义分镜）** 每个 shot 是否标注了景别+角度+主体+运镜
- [ ] **（可灵 3.0 多角色）** 台词是否标注了说话者角色（语气/语种/口音）
- [ ] **（可灵 3.0 长镜头）** 15s 长镜头是否按时间线组织动作变化

### 步骤1：TTS 台词配音

使用 edge-tts（微软神经网络语音，免费）或豆包 TTS（音质更好）为台词生成配音。

```bash
# 免费模式（edge-tts）
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_tts.py \
  --file 分镜数据.json --workspace /path/to/项目/ --engine free

# 豆包模式（音质更好，需配置 API Key）
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_tts.py \
  --file 分镜数据.json --workspace /path/to/项目/ --engine doubao

# 强制重新生成
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_tts.py \
  --file 分镜数据.json --workspace /path/to/项目/ --engine doubao --force
```

角色语音配置：
- **AI执政官**：高冷沉稳 2.0（豆包）/ `zh-CN-XiaoxiaoNeural`（免费）
- **臻叔**：云舟 2.0 讲故事风格（豆包）/ `zh-CN-YunjianNeural`（免费）

豆包 TTS 配置：在 `~/.ai-comic-drama/config.json` 中设置 `doubao_tts_api_key`

输出：`output_audio/tts/镜头XXX_台词.mp3` + `生成报告.json`

### 步骤2：音效生成

基于本地音效库，将中文音效描述自动匹配并混音。

```bash
# 初始化音效库目录（首次使用）
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_sfx.py --init-library

# 批量生成音效
python3 ~/.claude/skills/ai-comic-drama/scripts/generate_sfx.py \
  --file 分镜数据.json --workspace /path/to/项目/
```

音效库路径：`~/.ai-comic-drama/sfx_library/`，按 5 类组织：
- `ambient/` — 环境音（雨声、风声、脚步声）
- `mechanical/` — 机械音（金属撞击、摩擦、吱呀声）
- `electronic/` — 科技音（电子音、蜂鸣、启动声）
- `action/` — 动作音（蒸汽、心跳、键盘、电源键）
- `climax/` — 高潮/结局音（爆炸、碎裂、钢琴、鸟鸣）

用户需自行将 WAV/MP3 音效文件放入对应目录。

从分镜 JSON 的台词和时间数据自动生成 SRT 字幕。


### 步骤4：BGM 选择

用户自行提供 BGM 音频文件（如从 Pixabay/Mixkit 获取免费音乐），放入：

### 步骤5：视频合成

将视频片段 + TTS + 音效 + BGM + 字幕合成为最终成片。

```bash

  --file 分镜数据.json --workspace /path/to/项目/ \
  --video-dir ./output_videos/ \
  --tts-dir ./output_audio/tts/ \
  --sfx-dir ./output_audio/sfx/ \
  --bgm ./output_audio/bgm.mp3 \
  --subtitle ./output_audio/subtitle.srt \
  --output ./output_videos/最终成片.mp4

# 跳过 BGM 和字幕
python3 ~/.claude/skills/ai-comic-drama/scripts/assemble_video.py \
  --file 分镜数据.json --workspace /path/to/项目/ \
  --no-bgm --no-subtitle
```

合成流程包含 Doctor 检查：自动验证素材完整性、时长匹配，音频超长时自动变速不截断。
